## Завдання

Реалізувати голосового агента — помічника піцерії. Агент веде голосовий чат з
клієнтом і допомагає вибрати страву, оформити та відстежити замовлення.

Агент повинен вміти:

- показати меню або конкретну категорію (піца, напої, десерти)
- розказати деталі про страву — склад, ціну, наявність
- прийняти замовлення — зібрати позиції, ім'я, телефон та адресу
- перевірити статус замовлення за номером

Готовий файл `tools.py` (у нашому випадку — `fake_api.py`) з mock-даними та
реалізацією всіх функцій додається до завдання.

## Технічні вимоги

- **LiveKit Agents** як основний фреймворк (Worker + AgentSession)
- **OpenAI Realtime API** (модель `gpt-realtime-mini`) для голосової взаємодії — **обов'язково**
- Підключити tools із наданого `fake_api.py` через **function calling**
- Конфігурація через змінні середовища (`.env`)

## Tools API (з файлу `fake_api.py`)

Усі чотири функції вже реалізовані — потрібно лише підключити їх до агента.

- `get_menu(category?)` — повертає список позицій меню
  - `category`: `'pizza' | 'drinks' | 'desserts' | None`
  - повертає: `[{id, name, price, available, category}, ...]`
- `get_item_details(item_id)` — повна інформація про страву
  - `item_id`: рядок, наприклад `'pz1'`
  - повертає: `{id, name, description, price, size_cm, available}`
- `create_order(items, customer_name, phone, address)` — оформлює замовлення
  - `items`: `[{id: 'pz1', quantity: 2}, ...]`
  - повертає: `{success, order_id, items, total, estimated_minutes}`
- `get_order_status(order_id)` — статус замовлення
  - `order_id`: рядок, наприклад `'ORD-101'`
  - повертає: `{success, order_id, status, items, total}`

Тестові замовлення в mock-даних: **ORD-101** (статус «Готується»), **ORD-102** (статус «Їде до вас»).

## На що звертаємо увагу

- Агент запускається і відповідає в голосовому чаті
- Tools викликаються в потрібний момент і повертають коректний результат
- Діалог природний: короткі репліки, **без markdown у відповідях**
- Код зрозумілий і структурований
